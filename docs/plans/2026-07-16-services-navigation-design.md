# サービス導線再構成 設計

## 背景

トップページではNOVAのみを「PRODUCT」として紹介している一方、JQITには「AI導入伴走支援」という別のサービスサイトがある。また、ページ末尾の大きな問い合わせセクションは、共通ヘッダーのお問い合わせ導線と重複している。

## 決定事項

- 共通ヘッダーの「製品 / Product」を「サービス / Service」へ変更する
- 「サービス」はPCではドロップダウン、モバイルでは常時展開のサブリンクとする
- 配下に「NOVA」と「AI導入伴走支援」を同じ階層で掲載し、どちらも外部サイトを新しいタブで開く
- 親の「サービス」はトップページの `#service` へ移動する
- トップページの「PRODUCT」セクションを「SERVICES」へ変更し、2サービスを同じ重要度のカードで紹介する
- NOVAはSES営業支援プロダクトとして、AI導入伴走支援は既存業務のAI自動化支援として要点を分ける
- ページ末尾のCONTACTセクションを削除し、お問い合わせ導線は共通ヘッダーに集約する
- フッターにもAI導入伴走支援への外部リンクを追加する

## ページUI

サービスセクションは、セクション見出しと2枚のカードで構成する。

- 375px〜タブレットでは1列、1024px以上では2列にする
- 両カードは同じ余白、見出し階層、説明、3つの要点、外部リンクCTAを持つ
- NOVAは既存の管理画面画像を使う
- AI導入伴走支援は追加画像を作らず、既存のブランドカラーとタイポグラフィで専用ビジュアルを作る
- 外部リンクは `↗` を表示し、キーボードフォーカスを維持する
- 既存の赤・黒・クリームのブランドトーンを維持する

## ナビゲーション

- PC: 「サービス」にフォーカスまたはホバーすると2リンクを表示する
- モバイル: 「サービス」の下に2リンクを常時表示する
- 外部リンクには `target="_blank"` と `rel="noopener noreferrer"` を付ける
- 外部リンクの矢印は内部遷移の `→` ではなく `↗` とする

## 検証

- ヘッダーとモバイルメニューに「サービス」「NOVA」「AI導入伴走支援」が表示される
- トップページの `#service` に2サービスが表示され、両CTAが正しい外部URLを開く
- CONTACTセクションがトップページから消えている
- 375px、768px、1024px、1440pxで文字切れ・横スクロール・タップ領域を確認する
- Nodeテスト、lint、本番ビルドを通す
