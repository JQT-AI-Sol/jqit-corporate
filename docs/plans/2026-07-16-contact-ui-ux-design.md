# お問い合わせ導線・フォーム UI/UX 改善設計

## 背景

社内レビューで、以下の課題が確認された。

- スマートフォンでお問い合わせフォーム入力中にプライバシーポリシーへ遷移すると、入力内容が失われる
- スマートフォンで文章の改行、文字サイズ、ボタンサイズなどが最適化されていない
- PC表示でお問い合わせ導線が多く、ページ内の優先順位が分かりにくい
- 実績情報が不足しているが、掲載内容は先に社内で整理する必要がある

本設計では、実績コンテンツの追加は行わず、フォームと問い合わせ導線のUI/UX改善のみを対象とする。

## 問い合わせ導線

お問い合わせへの主要導線は、全ページ共通ヘッダーとトップページ最下部の問い合わせセクションに限定する。

- PCヘッダーのお問い合わせボタンを残す
- モバイルメニュー内のお問い合わせボタンを残す
- トップページ最下部の問い合わせセクションを残す
- トップページのファーストビューからお問い合わせボタンを削除する
- トップページの協業セクションから「協業のご相談」ボタンを削除する
- 会社情報、ビジョン、事業詳細、ニュースなど下層ページ内の問い合わせCTAを削除する
- フッターのお問い合わせリンクを削除する
- お問い合わせページとプライバシーポリシー内の問い合わせ窓口は、機能上必要なため維持する

これにより、問い合わせ先は常にヘッダーから見つけられる一方、各ページの閲覧中に同じCTAが繰り返し現れる状態を解消する。

## フォーム入力内容の保持

通常公開用フォームと静的公開用フォームの両方を同じ仕様にする。

- プライバシーポリシーは別タブで開き、フォーム画面を維持する
- 入力内容はブラウザのセッションストレージへ一時保存する
- お問い合わせページへ戻った際は保存内容を復元する
- 送信成功時は保存内容を削除する
- 同意チェックは復元対象に含めず、送信時に改めて同意を求める
- 保存データは当該タブのセッション内だけで利用し、永続保存しない

## スマートフォン表示

- 入力欄と選択欄の文字サイズを16px以上にし、iOSの自動ズームを防ぐ
- 入力欄、チェックボックス周辺、送信ボタンのタップ領域を44px以上にする
- 送信ボタンはスマートフォンで横幅いっぱいにし、600px以上では内容幅に戻す
- フォームの説明文とラベルの行間を読みやすく調整する
- 長い日本語文言が不自然に分断されないよう、改行単位を調整する
- 375px、390px、430px、768px、1024px、1440pxで表示を確認する

## エラー処理とアクセシビリティ

- 現在の項目別エラー表示と `aria-invalid` / `aria-describedby` を維持する
- キーボードフォーカス表示を維持する
- 送信中はボタンを無効化し、状態文言を表示する
- セッションストレージが利用できない環境でもフォーム入力と送信は継続できるようにする

## 検証

- 通常公開用フォームと静的公開用フォームで入力保存・復元・送信成功後の削除を確認する
- プライバシーポリシーが別タブで開くことを確認する
- PCでヘッダーとトップページ最下部以外の主要問い合わせCTAが表示されないことを確認する
- スマートフォンで横スクロールがなく、入力欄とボタンが操作可能な大きさであることを確認する
- lint、型チェック、本番ビルドを通す

## 実績情報（別対応）

実績情報は実装前に、公開可否を含めて次の情報を整理する。

- 顧客名または匿名表記
- 業界・企業規模
- 導入前の課題
- 提供サービスと対応範囲
- 実施期間・体制
- 定量成果・定性成果
- 掲載可能なロゴ、画面、写真
- 顧客コメント
- 守秘義務・公開許諾の確認状況
